## 실무 교육용 슬라이드: 마크다운 프레젠테이션

다음은 **'마크다운으로 슬라이드 프레젠테이션 만들기'**를 주제로 한 20페이지 분량의 실무 교육용 슬라이드입니다. 마크다운의 기본 개념부터 Pandoc, Marp 등 주요 도구를 활용한 변환 및 고급 기능까지 체계적으로 다루어 초보자도 쉽게 학습할 수 있도록 구성했습니다.

# 마크다운 프레젠테이션
## 텍스트로 만드는 효율적인 슬라이드 제작법
- **마크다운(Markdown)을 활용한 새로운 방식의 프레젠테이션 제작 방법 소개**
- 텍스트 기반 문서 작성의 단순함과 다양한 변환 도구의 강력함 결합
- 개발자, 연구원, 학생 등 문서 작업 효율을 높이고 싶은 모두를 위한 기술
- 복잡한 UI 없이 내용에만 집중하여 세련된 슬라이드를 만드는 과정 학습
- 최종 목표는 Pandoc, Marp 등 도구를 활용해 실무용 슬라이드를 제작하는 것

::: notes
안녕하세요. 오늘 교육을 맡은 슬라이드 제작 전문가입니다. 이번 시간에는 마크다운(Markdown)을 이용해 효율적으로 프레젠테이션 자료를 만드는 방법을 알아보겠습니다. 기존의 파워포인트나 키노트와는 다른, 텍스트 중심의 새로운 슬라이드 제작 워크플로우를 배우게 될 것입니다. 마크다운의 간결함과 Pandoc, Marp와 같은 강력한 변환 도구를 결합하여, 내용에만 집중하면서도 전문가 수준의 결과물을 만드는 방법을 목표로 합니다.
:::

# 왜 마크다운인가?
## 마크다운 프레젠테이션의 장점
- **작성이 쉽고 가독성이 높아 내용에만 집중할 수 있는 환경 제공**
- Git과 같은 버전 관리 시스템(VCS)을 통해 변경 이력 관리가 용이함
- 텍스트 파일이므로 거의 모든 편집기에서 작성 및 수정이 가능함
- Pandoc, Marp 등의 도구를 통해 HTML, PDF, PPTX 등 다양한 포맷으로 손쉬운 변환
- 복잡한 서식 지정 없이 구조화된 콘텐츠를 빠르게 생산할 수 있는 장점

```
<비교 인포그래픽>
기존 방식: [디자인 도구] -> [콘텐츠 입력] -> [서식 조정] -> [완성]
마크다운:   [콘텐츠 작성] -> [명령어 변환] -> [완성]
```
::: notes
프레젠테이션을 만들 때 왜 마크다운을 사용해야 할까요? 가장 큰 장점은 **단순함과 효율성**입니다. 보시는 것처럼 마크다운은 LaTeX 같은 다른 마크업 언어에 비해 훨씬 간결합니다. 또한 일반 텍스트 파일이라 Git으로 버전을 관리하기 매우 편리하며, Pandoc이나 Marp 같은 도구를 사용하면 단 하나의 소스 파일로 HTML, PDF, 파워포인트 등 다양한 결과물을 만들 수 있습니다. 덕분에 디자인 고민 없이 내용에만 집중할 수 있습니다.
:::

# 핵심 원칙
## 마크다운 슬라이드 작성의 기본 규칙
- **하나의 마크다운(.md) 파일에 모든 슬라이드의 내용을 순서대로 작성함**
- **가로줄(---) 세 개를 사용하여 각 슬라이드 페이지를 명확하게 구분함**
- **가장 큰 제목(H1, #)을 사용하여 각 슬라이드의 제목을 정의하는 것이 일반적임**
- 문서 상단에 YAML Front Matter를 사용해 제목, 작성자, 테마 등 메타데이터 설정
- 텍스트 편집기만 있으면 언제 어디서든 슬라이드 내용 수정 및 관리가 가능함

<br>
**[실무 예시: 기본 슬라이드 구조]**
```markdown
---
title: "발표 제목"
author: "발표자"
---

# 1번 슬라이드
- 내용 1

---

# 2번 슬라이드
- 내용 2
```
::: notes
마크다운으로 슬라이드를 만들 때 기억해야 할 몇 가지 핵심 원칙이 있습니다. 첫째, 모든 슬라이드 내용은 단 하나의 마크다운 파일에 담습니다. 둘째, 슬라이드와 슬라이드 사이는 세 개의 대시(`---`)로 구분합니다. 셋째, 일반적으로 `#`을 사용한 최상위 제목이 슬라이드 제목이 됩니다. 마지막으로, 문서의 맨 앞에 YAML 형식의 'Front Matter'를 추가하여 발표 전체의 제목, 작성자, 날짜, 테마 같은 정보를 설정할 수 있습니다. 이 예시처럼 구조가 매우 직관적입니다.
:::

# 필수 마크다운 문법
## 슬라이드 제작을 위한 기본기
- **제목(Headers):** `#`의 개수로 수준을 조절하여 슬라이드 제목과 소제목을 구성함
- **목록(Lists):** 순서 없는 목록(`*`, `-`)과 순서 있는 목록(`1.`)으로 내용 구조화
- **강조(Emphasis):** `*Italic*`, `**Bold**` 등을 사용해 핵심 단어나 문장을 강조함
- **코드 블록(Code Blocks):** ``` 기호를 사용해 소스 코드나 명령어 예시를 깔끔하게 표시
- **링크 및 이미지(Links & Images):** `[텍스트](URL)` 형식으로 하이퍼링크나 이미지를 손쉽게 삽입

| 문법 종류 | 마크다운 예시 | 결과물 |
| :--- | :--- | :--- |
| 제목 2 | `## 소제목` | ## 소제목 |
| 강조 | `**중요한 내용**` | **중요한 내용** |
| 인라인 코드 | `` `code()` `` | `code()` |
| 링크 | `[Pandoc](https://pandoc.org)` | [Pandoc](https://pandoc.org) |

::: notes
슬라이드 제작에 필요한 최소한의 마크다운 문법입니다. 다행히 매우 간단합니다. `#` 기호로 제목의 수준을 정하고, 별표나 숫자로 목록을 만듭니다. 별표 한두 개로 텍스트를 감싸서 기울이거나 굵게 만들 수 있고, 백틱(```)을 이용해 코드 블록을 만들면 가독성이 높아집니다. 보시는 표와 같이 몇 가지 기본 규칙만 익히면 누구나 구조적인 문서를 작성할 수 있습니다.
:::

# 주요 변환 도구
## 마크다운을 슬라이드로 만드는 방법들
- **Pandoc:** 거의 모든 문서 포맷을 변환할 수 있는 '만능 문서 변환기'
- **Marp:** VS Code 확장 기능과 CLI를 제공하는 마크다운 프레젠테이션 생태계
- **Reveal.js:** 강력한 기능을 갖춘 HTML 기반 프레젠테이션 프레임워크
- **Lookatme:** 개발자에게 친숙한 터미널 환경에서 슬라이드를 보여주는 도구
- **Slidev:** Vue.js 기반으로 웹 개발자에게 최적화된 차세대 슬라이드 도구

| 도구 | 주요 특징 | 출력 형식 | 대상 사용자 |
| :--- | :--- | :--- | :--- |
| **Pandoc** | 범용성, 강력한 확장성 | HTML, PDF, PPTX 등 | 학술, 기술 문서 작성자 |
| **Marp** | 사용 편의성, 실시간 미리보기 | HTML, PDF, PPTX | 모든 사용자, 특히 초보자 |
| **Reveal.js** | 기능성, 동적 효과, 2D 슬라이드 | HTML | 웹 개발자, 고급 사용자 |
| **Lookatme** | 터미널 기반, 단순함 | 터미널 출력 | 터미널 환경 선호 개발자 |

::: notes
작성된 마크다운 파일을 실제 슬라이드로 변환해주는 여러 도구가 있습니다. **Pandoc**은 거의 모든 문서 형식을 지원하는 가장 강력하고 범용적인 도구입니다. **Marp**는 VS Code와 통합되어 실시간 미리보기를 제공해 초보자도 쉽게 사용할 수 있습니다. **Reveal.js**는 화려한 효과와 2D 슬라이드 같은 고급 기능을 제공하는 HTML 프레임워크입니다. 이 외에도 터미널에서 바로 발표할 수 있는 **Lookatme** 같은 도구도 있습니다. 각 도구의 특징을 비교해보고 자신에게 맞는 것을 선택하는 것이 중요합니다.
:::

# Pandoc
## 만능 문서 변환기
- **하나의 소스로 다양한 결과물 생성:** HTML, PDF, PPTX 등 30개 이상의 포맷 지원
- **명령줄 인터페이스(CLI) 기반:** 스크립트를 통한 자동화된 문서 변환 워크플로우 구축 가능
- **강력한 확장 기능:** 필터(Filter)와 템플릿(Template)을 이용해 출력물을 세밀하게 제어
- **학술적 글쓰기에 최적화:** 각주, 인용, 참고 문헌 자동 생성 기능 지원
- **다양한 슬라이드 프레임워크 지원:** Reveal.js, Beamer, Slidy 등과 연동하여 슬라이드 제작

```
<변환 과정 인포그래픽>
[Markdown 파일] --(Pandoc)--> [AST] --(Writer)--> [HTML/PDF/PPTX]
```
::: notes
이제 각 도구를 좀 더 자세히 살펴보겠습니다. Pandoc은 '문서계의 스위스 군용 칼'이라 불릴 만큼 강력한 변환 도구입니다. 마크다운으로 작성된 파일을 HTML 슬라이드, LaTeX 기반의 고품질 PDF, 심지어 파워포인트 파일로도 변환할 수 있습니다. 내부적으로는 입력 문서를 '추상 구문 트리(AST)'라는 중간 형태로 변환한 뒤, 원하는 출력 형식으로 다시 쓰는 모듈식 설계를 가집니다. 이를 통해 엄청난 유연성을 확보합니다.
:::

# Pandoc 변환 실습
## HTML 및 PowerPoint 슬라이드 만들기
- **HTML 슬라이드 변환:** `-t` 옵션으로 `revealjs`, `slidy` 등 원하는 형식을 지정함
- **독립 실행형 파일:** `-s` 또는 `--standalone` 옵션으로 완전한 HTML 문서를 생성함
- **PowerPoint 변환:** `-o` 옵션으로 출력 파일 확장자를 `.pptx`로 지정하면 자동으로 변환됨
- **의존성 없는 파일:** `--self-contained` 옵션으로 CSS, JS, 이미지를 파일 내에 모두 포함시킴
- **다중 파일 입력:** 여러 마크다운 파일을 순서대로 입력하여 하나의 프레젠테이션으로 병합 가능

<br>
**[실무 예시: 변환 명령어]**
```bash
# Reveal.js HTML 슬라이드 생성
pandoc slides.md -t revealjs -s -o presentation.html

# PowerPoint(.pptx) 파일 생성
pandoc slides.md -o presentation.pptx

# 모든 리소스를 포함한 단일 HTML 파일 생성
pandoc slides.md -t slidy -s --self-contained -o presentation.html
```
::: notes
Pandoc 사용법은 간단합니다. 터미널에서 `pandoc` 명령어를 사용합니다. 이 예시 코드처럼 `-t revealjs`는 출력을 Reveal.js 형식으로, `-s`는 웹 브라우저에서 바로 열 수 있는 완전한 HTML 파일을 만들라는 의미입니다. 출력 파일명을 `-o` 옵션으로 지정하는데, 확장자를 `.pptx`로 바꾸기만 하면 파워포인트 파일이 생성됩니다. `--self-contained` 옵션을 추가하면 이미지나 스타일시트 같은 외부 파일 없이 단일 파일로 배포할 수 있어 매우 유용합니다.
:::

# Pandoc PDF 제작
## LaTeX Beamer 연동
- **고품질 PDF 생성:** Pandoc은 LaTeX Beamer를 통해 전문적인 조판 품질의 PDF 슬라이드를 제작함
- **LaTeX 설치 필수:** 이 기능을 사용하려면 시스템에 `pdflatex` 같은 LaTeX 엔진이 설치되어야 함
- **테마 적용:** `-V theme=Warsaw`와 같이 변수(`-V`)를 이용해 다양한 Beamer 테마를 적용함
- **PDF 옵션 제어:** `documentclass`, `fontsize` 등 LaTeX 관련 변수로 세부 설정 제어 가능
- **디버깅 팁:** PDF 변환 실패 시, `-o test.tex`로 중간 LaTeX 파일을 생성해 오류를 확인할 수 있음

```
<변환 과정 인포그래픽>
[slides.md] -> [pandoc -t beamer] -> [slides.tex] -> [pdflatex] -> [slides.pdf]
```
::: notes
학술 발표나 공식 문서처럼 고품질의 PDF 슬라이드가 필요할 때 Pandoc은 LaTeX Beamer를 활용합니다. Beamer는 LaTeX 기반의 프레젠테이션 클래스로, 수식이나 참고 문헌 표현이 탁월합니다. 사용법은 간단히 `-t beamer` 옵션을 주면 되지만, 시스템에 LaTeX 배포판이 설치되어 있어야 합니다. `-V` 옵션으로 `theme`, `colortheme` 같은 변수를 설정해 슬라이드의 디자인을 손쉽게 변경할 수 있습니다.
:::

# Marp
## 통합형 프레젠테이션 생태계
- **Markdown Presentation Ecosystem:** 마크다운으로 슬라이드를 만드는 데 필요한 모든 것을 제공함
- **VS Code 확장 기능:** 실시간 미리보기를 지원하여 직관적인 슬라이드 작성이 가능함
- **간편한 내보내기:** 클릭 몇 번으로 HTML, PDF, PPTX 형식의 파일을 손쉽게 생성
- **기본 테마 제공:** `default`, `gaia`, `uncover` 세 가지의 세련된 기본 테마를 내장함
- **CSS 기반 커스터마이징:** 자신만의 CSS 파일을 만들어 테마를 확장하거나 새로 만들 수 있음

<br>
**[실무 예시: VS Code에서의 Marp]**
- 좌측에는 마크다운 에디터, 우측에는 실시간으로 렌더링된 슬라이드 미리보기가 표시됨.
- 코드 변경 시 즉시 미리보기에 반영되어 생산성이 매우 높음.

::: notes
Pandoc이 강력하지만 설정이 다소 복잡하게 느껴진다면 Marp가 훌륭한 대안입니다. Marp는 '마크다운 프레젠테이션 생태계'라는 이름처럼, 슬라이드 제작에 필요한 모든 기능을 통합적으로 제공합니다. 특히 VS Code 확장 기능은 강력 추천하는데, 코드를 작성하는 동시에 실제 슬라이드 모습을 바로 옆에서 확인할 수 있어 생산성이 극대화됩니다. 작성 완료 후에는 클릭 한 번으로 PDF나 PPT 파일로 내보낼 수 있습니다.
:::

# Marp 기본 사용법
## 첫 슬라이드 만들기
- **Marp 활성화:** YAML Front Matter에 `marp: true`를 추가하여 Marp 렌더링 엔진을 활성화함
- **슬라이드 나누기:** `---` 구분자를 사용하여 새로운 슬라이드 페이지를 생성함
- **콘텐츠 작성:** 표준 마크다운 문법을 그대로 사용하여 슬라이드 내용을 자유롭게 작성
- **실시간 미리보기:** VS Code에서 미리보기 창을 열어 작업 결과를 즉시 확인
- **테마 지정:** `theme: gaia`와 같이 Front Matter에 테마 이름을 지정하여 디자인 적용

<br>
**[실무 예시: 최소 Marp 문서]**
```markdown
---
marp: true
theme: gaia
---

# 첫 번째 슬라이드
- Marp는 매우 간단합니다.

---

# 두 번째 슬라이드
- 슬라이드는 --- 로 구분합니다.
```
::: notes
Marp 사용법은 매우 직관적입니다. 문서 최상단에 `---`로 감싼 영역을 만들고, 그 안에 `marp: true`라고 적어주기만 하면 됩니다. 그러면 VS Code의 미리보기 엔진이 Marp 모드로 전환됩니다. 그 후로는 표준 마크다운 문법을 사용해 내용을 채우고, 슬라이드를 나눌 때 `---`를 사용하면 됩니다. `theme` 속성을 추가해 내장된 `gaia`나 `uncover` 테마를 적용할 수도 있습니다.
:::

# Marp 테마와 스타일링
## 슬라이드 디자인 맞춤 설정
- **기본 테마:** `default`, `gaia`, `uncover` 세 가지 기본 테마를 즉시 사용 가능함
- **색상 반전:** `class: invert` 지시자를 추가하여 어두운 배경의 반전 테마를 쉽게 적용함
- **사용자 정의 CSS:** 나만의 `.css` 파일을 만들어 `theme` 속성에 지정하여 사용할 수 있음
- **인라인 스타일:** `<style>` 태그를 마크다운 내에 직접 삽입하여 특정 요소의 스타일을 수정함
- **글로벌 지시자:** Front Matter에 `class`, `paginate` 등을 설정하여 전체 슬라이드에 일괄 적용

| 지시자(Directive) | 설명 | 예시 |
| :--- | :--- | :--- |
| `theme` | 사용할 테마를 지정함 | `theme: uncover` |
| `class` | 슬라이드 전체에 적용할 클래스 | `class: invert` |
| `paginate` | 슬라이드에 페이지 번호를 표시함 | `paginate: true` |
| `backgroundColor` | 전체 슬라이드의 배경색 지정 | `backgroundColor: #f0f0f0`|
| `backgroundImage` | 전체 슬라이드의 배경 이미지 지정 | `backgroundImage: url(...)` |

::: notes
Marp는 디자인 커스터마이징이 매우 유연합니다. 기본 제공되는 3가지 테마만으로도 훌륭하지만, 더 나아가 `class: invert`를 추가해 쉽게 다크 모드로 전환할 수 있습니다. 더 세밀한 조정을 원한다면, CSS 파일을 직접 만들어 `theme`로 지정하거나, 슬라이드 내에 `<style>` 태그를 직접 삽입해 특정 부분만 바꿀 수도 있습니다. 표에 보이는 것처럼 Front Matter에 다양한 지시자를 추가해 페이지 번호나 배경색 등을 전역으로 설정할 수도 있습니다.
:::

# Marp 고급 스타일링
## 슬라이드별 디자인 제어
- **지역(Local) 지시자:** 특정 슬라이드에만 스타일을 적용하기 위해 주석 문법을 활용함
- **단일 슬라이드 스타일:** `<!-- _class: lead -->`는 해당 슬라이드에만 `lead` 클래스를 적용함
- **이후 모든 슬라이드 스타일:** `<!-- class: invert -->`는 현재 슬라이드부터 끝까지 `invert` 클래스 적용
- **HTML 활성화:** VS Code 설정에서 HTML 렌더링을 활성화하면 `<div>`, `<span>` 등 사용 가능
- **배경 이미지 제어:** `![bg left:40%]`와 같이 이미지 위치, 크기, 필터 효과 등을 개별 지정

<br>
**[실무 예시: 특정 슬라이드 강조]**
```markdown
---
<!-- _class: lead gaia -->

## 📌 옴니보어: 다양한 취향의 '잡식성 소비자'
현대 소비자는 세대, 계층, 성별에 묶이지 않습니다.
```
::: notes
전체 슬라이드가 아닌 특정 슬라이드만 다르게 꾸미고 싶을 때도 있습니다. Marp는 이를 위해 주석 형태의 '지역 지시자'를 제공합니다. 예를 들어, `<!-- _class: lead -->`라고 적으면 현재 슬라이드에만 `lead` 스타일(보통 텍스트를 크게 중앙 정렬)이 적용됩니다. `_`가 빠진 `<!-- class: invert -->`는 그 지점부터 끝까지 스타일을 변경합니다. 또한, `![bg]` 태그에 `left:40%` 같은 옵션을 추가하여 슬라이드마다 배경 이미지를 다르게 설정할 수도 있습니다.
:::

# 시각 자료 활용
## 이미지와 배경 삽입
- **기본 이미지 삽입:** 표준 마크다운 `![대체 텍스트](경로)` 문법을 그대로 사용함
- **배경 이미지:** Marp의 특별한 `![bg](경로)` 태그를 사용해 슬라이드 전체 배경을 설정함
- **배경 위치 조정:** `![bg left]` 또는 `![bg right]` 키워드로 배경 이미지의 좌우 정렬
- **배경 크기 조절:** `![bg left:35%]`처럼 비율을 지정하여 이미지 크기를 세밀하게 조정함
- **이미지 필터 효과:** `grayscale`(흑백), `blur`(흐림), `opacity`(투명도) 등 다양한 필터 적용

| `![bg]` 태그 옵션 | 설명 |
| :--- | :--- |
| `right` / `left` | 이미지를 오른쪽 또는 왼쪽에 정렬함 |
| `:X%` | 이미지 크기를 슬라이드 너비의 X%로 설정함 |
| `grayscale` | 이미지를 흑백으로 변환함 |
| `blur` | 이미지에 블러 효과를 적용함 |
| `opacity` | 이미지의 투명도를 조절함 |

::: notes
프레젠테이션에서 이미지는 매우 중요합니다. 일반적인 이미지는 표준 마크다운 문법으로 넣으면 됩니다. 슬라이드 배경을 이미지로 채우고 싶을 때는 Marp의 강력한 `![bg]` 태그를 사용하세요. 이 태그는 단순히 이미지를 채우는 것을 넘어, `left`, `right` 키워드로 위치를 지정하거나, `left:35%`처럼 크기를 비율로 조절할 수도 있습니다. `blur`나 `opacity` 같은 필터를 적용해 텍스트 가독성을 높이는 것도 좋은 방법입니다.
:::

# 시각 자료 활용
## Mermaid 다이어그램
- **텍스트 기반 다이어그램:** 코드를 작성하듯 순서도, 시퀀스 다이어그램 등을 생성하는 라이브러리
- **Marp와의 통합:** Marp는 Mermaid 문법을 자동으로 인식하여 다이어그램 이미지로 변환함
- **간단한 사용법:** `<div class="mermaid">` 태그로 Mermaid 코드를 감싸 슬라이드에 추가함
- **복잡한 개념 시각화:** 코드의 흐름이나 시스템 아키텍처를 설명할 때 매우 유용함
- **스타일 커스터마이징:** CSS를 통해 다이어그램의 크기, 색상, 폰트 등을 자유롭게 변경 가능

<br>
**[실무 예시: 간단한 순서도]**
```html
<div class="mermaid">
graph TD;
    A[사용자 입력] --> B{데이터 처리};
    B --> C[결과 출력];
    B --> D[오류 처리];
</div>
```
::: notes
복잡한 프로세스나 관계를 설명할 때 다이어그램은 필수적입니다. Mermaid는 텍스트로 다이어그램을 그릴 수 있게 해주는 도구로, Marp와 완벽하게 통합됩니다. 예시처럼 `graph TD` 같은 간단한 코드를 작성하면, Marp가 이를 자동으로 멋진 순서도로 변환해줍니다. 개발 프로세스, 시스템 구성도, 데이터 흐름 등을 설명할 때 파워포인트에서 도형을 그리는 것보다 훨씬 빠르고 효율적입니다.
:::

# Reveal.js
## 강력한 HTML 프레젠테이션 프레임워크
- **HTML 기반 프레임워크:** 웹 기술을 최대한 활용하여 동적이고 인터랙티브한 슬라이드 제작
- **2D 슬라이드 구조:** 좌우(수평) 이동뿐만 아니라 상하(수직) 이동이 가능한 입체적 구조 지원
- **풍부한 기능:** 다양한 테마, 전환 효과, 코드 하이라이팅, 플러그인 생태계 보유
- **Pandoc 연동:** `pandoc -t revealjs` 명령어로 마크다운을 Reveal.js 슬라이드로 쉽게 변환
- **고급 커스터마이징:** HTML 주석을 이용해 슬라이드별 배경, 전환 효과 등 세부 속성 제어

```
<2D 슬라이드 구조 인포그래픽>

[ 1.1 ] <-> [ 1.2 ] <-> [ 1.3 ]
             ^
             |
             v
           [ 2.2 ]
             ^
             |
             v
           [ 3.2 ]
```
::: notes
최고 수준의 기능과 유연성을 원한다면 Reveal.js가 정답입니다. Reveal.js는 HTML 기반 프레임워크로, 웹에서 구현할 수 있는 거의 모든 동적 효과를 슬라이드에 담을 수 있습니다. 가장 큰 특징은 보시는 것처럼 상하좌우로 이동하는 2차원 슬라이드 구조입니다. 대주제는 수평으로, 소주제는 수직으로 구성하여 발표의 흐름을 입체적으로 만들 수 있습니다. Pandoc을 이용하면 마크다운으로 작성한 내용을 손쉽게 Reveal.js 슬라이드로 변환할 수 있습니다.
:::

---

# Reveal.js와 Pandoc 연동
## 마크다운으로 Reveal.js 슬라이드 만들기
- **변환 명령어:** `pandoc -t revealjs -s -o slides.html slides.md` 형식으로 변환
- **테마 적용:** `-V theme=sky`처럼 `-V` 옵션을 사용하여 내장된 테마(black, white, sky 등)를 적용
- **전환 효과:** `-V transition=zoom` 옵션으로 슬라이드 전환 효과(fade, slide, zoom 등)를 설정
- **외부 리소스 연결:** 기본적으로 웹(CDN)에 있는 Reveal.js 파일을 연결하여 사용함
- **슬라이드 레벨 지정:** `--slide-level=2` 옵션으로 H1을 수평, H2를 수직 슬라이드로 지정

| Pandoc 옵션 | 설명 | 예시 값 |
| :--- | :--- | :--- |
| `-t revealjs` | 출력 포맷을 Reveal.js로 지정함 | - |
| `-V theme` | 슬라이드 테마를 설정함 | `sky`, `moon`, `serif` |
| `-V transition` | 전환 효과를 설정함 | `zoom`, `fade`, `convex` |
| `--slide-level` | 슬라이드 계층 구조를 결정함 | `2` (H1 수평, H2 수직) |
| `--self-contained`| 모든 리소스를 단일 파일에 포함 | - |

::: notes
Reveal.js를 직접 HTML로 작성하는 것은 복잡할 수 있지만, Pandoc을 사용하면 매우 간단해집니다. `pandoc -t revealjs`라는 명령어 하나면 마크다운 문서가 Reveal.js 슬라이드로 변환됩니다. 표에서 보시는 것처럼 `-V` 옵션을 사용해 `theme`이나 `transition` 같은 Reveal.js의 다양한 설정을 변수 형태로 전달할 수 있습니다. 예를 들어, `theme`을 'sky'로, `transition`을 'zoom'으로 설정하여 발표 자료에 개성을 더할 수 있습니다.
:::

# 핵심 기능: 발표자 노트
## 청중에게는 보이지 않는 나만의 대본
- **발표 지원:** 슬라이드 내용 외에 발표자가 참고할 추가 정보나 대본을 기록하는 기능
- **다양한 도구 지원:** Pandoc, Marp, Reveal.js 모두 각자의 방식으로 발표자 노트를 지원함
- **발표자 모드:** 발표 시 별도의 창(또는 화면)에 현재 슬라이드, 다음 슬라이드, 노트가 함께 표시됨
- **작성 편의성:** 마크다운 문법을 사용하여 노트 내용을 구조적으로 작성할 수 있음
- **자신감 향상:** 발표 흐름을 놓치지 않도록 도와주어 안정적인 발표를 가능하게 함

| 도구 | 발표자 노트 작성 문법 |
| :--- | :--- |
| **Pandoc** | `::: notes` ... `:::` |
| **Marp** | `<!-- ... -->` (HTML 주석) |
| **Reveal.js** | `<aside class="notes">` ... `</aside>` |

::: notes
발표자 노트는 성공적인 프레젠테이션을 위한 핵심 기능입니다. 청중에게는 슬라이드만 보이지만, 발표자는 별도의 화면에서 자신만의 대본을 볼 수 있습니다. 오늘 배운 Pandoc, Marp, Reveal.js 모두 발표자 노트를 지원하며, 표에서 보시는 것처럼 문법은 조금씩 다릅니다. Pandoc의 경우 제가 지금 사용하고 있는 것처럼 `::: notes` 블록을 사용합니다. 이 기능을 활용하면 키워드를 잊지 않고 발표의 흐름을 자연스럽게 이어갈 수 있습니다.
:::

# 핵심 기능: 다단 레이아웃
## 콘텐츠를 나란히 배치하기
- **콘텐츠 병렬 배치:** 이미지와 설명, 두 가지 코드 비교 등 내용을 나란히 보여줄 때 유용함
- **가독성 향상:** 관련 있는 정보를 한눈에 비교하며 볼 수 있어 이해도를 높임
- **Fenced Divs 활용:** Pandoc에서는 `:::`로 감싼 블록을 사용해 컬럼 구조를 생성함
- **클래스 지정:** 바깥쪽 Div에는 `columns` 클래스, 안쪽 Div들에는 `column` 클래스를 지정함
- **너비 조절:** 각 `column` Div에 `width="40%"`와 같이 너비를 직접 지정하여 비율을 조절함

<br>
**[실무 예시: 2단 컬럼 구조]**
```markdown
:::::::::::::: {.columns}
::: {.column width="40%"}
- 항목 1
- 항목 2
:::
::: {.column width="60%"}
![이미지](image.png)
:::
::::::::::::::
```
::: notes
슬라이드에 두 개 이상의 콘텐츠를 나란히 배치하고 싶을 때가 있습니다. Pandoc은 이를 위해 'Fenced Divs'라는 특별한 문법을 제공합니다. 예시 코드처럼 `.columns` 클래스를 가진 부모 Div 안에 `.column` 클래스를 가진 자식 Div들을 넣으면 됩니다. 각 `.column`에는 `width` 속성을 부여하여 너비 비율을 자유롭게 조절할 수 있습니다. 이를 통해 텍스트와 이미지를 나란히 두거나 두 개의 코드를 비교하는 등 다양한 레이아웃을 구현할 수 있습니다.
:::

# 실무 워크플로우
## 효율적인 슬라이드 제작 과정
- **1단계 (내용 구상):** 디자인 고민 없이 마크다운으로 발표의 구조와 핵심 내용부터 빠르게 작성함
- **2단계 (도구 선택):** 발표 목적과 환경에 맞춰 Pandoc, Marp 등 최적의 변환 도구를 선택함
- **3단계 (1차 변환):** 기본 옵션으로 슬라이드를 변환하여 전체적인 구조와 흐름을 먼저 확인함
- **4단계 (스타일링):** 내장 테마를 적용하거나, 간단한 CSS 수정을 통해 시각적 완성도를 높임
- **5단계 (최종 검토):** 발표자 노트를 추가하고, 내보내기(Export) 기능을 통해 최종 결과물을 생성함

```
<작업 흐름 인포그래픽>
[콘텐츠 작성 (MD)] -> [도구 선택] -> [기본 변환] -> [테마/CSS 적용] -> [최종 파일 생성]
```
::: notes
효율적인 실무 워크플로우를 제안합니다. 먼저, 파워포인트를 열지 말고 마크다운으로 내용의 뼈대부터 잡으세요. 내용이 완성되면, 발표의 성격에 따라 Marp의 간편함을 택할지, Pandoc의 강력함을 택할지 결정합니다. 그 다음, 기본 설정으로 한 번 변환해 전체 구조를 확인하고, 마지막으로 테마를 적용하거나 CSS를 살짝 수정해 디자인을 다듬습니다. 이 과정을 통해 디자인이 아닌 '내용' 중심의 생산적인 슬라이드 제작이 가능해집니다.
:::
